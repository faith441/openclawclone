# Report Generator

## Overview

Transforms raw data into structured, readable reports with executive summaries, key findings, and recommendations.

## When to Use

- User has data and needs a report
- User asks for a summary or analysis document
- User needs to present findings to stakeholders

## Report Template

```markdown
# [Report Title]
**Date:** [Date]
**Prepared by:** [Author]
**Period:** [Time range if applicable]

---

## Executive Summary

[2-3 paragraph overview of key findings and recommendations]

**Key Metrics:**
| Metric | Value | Change |
|--------|-------|--------|
| [Metric] | [Value] | [+/-X%] |

---

## Key Findings

### 1. [Finding Title]
[Description with supporting data]

### 2. [Finding Title]
[Description with supporting data]

### 3. [Finding Title]
[Description with supporting data]

---

## Detailed Analysis

### [Section 1]
[In-depth analysis with data tables and explanations]

### [Section 2]
[In-depth analysis with data tables and explanations]

---

## Recommendations

1. **[Recommendation]** - [Rationale and expected impact]
2. **[Recommendation]** - [Rationale and expected impact]
3. **[Recommendation]** - [Rationale and expected impact]

---

## Appendix

### Data Sources
- [Source 1]
- [Source 2]

### Methodology
[Brief description of how analysis was conducted]

### Raw Data
[Tables or references to detailed data]
```

## Report Types

| Type | Focus | Sections |
|------|-------|----------|
| Status Report | Progress update | Summary, Completed, In Progress, Blockers |
| Analytics Report | Data insights | Metrics, Trends, Anomalies, Recommendations |
| Incident Report | Issue analysis | Timeline, Impact, Root Cause, Resolution |
| Comparison Report | A vs B analysis | Criteria, Comparison Table, Verdict |

## Guidelines

1. **Lead with insights** - Most important findings first
2. **Use visuals** - Tables, charts where appropriate
3. **Be specific** - Include actual numbers and dates
4. **Actionable recommendations** - What should happen next
