# Getting Started with Zenthral SaaS

## 🎉 Congratulations!

You now have a **complete multi-tenant SaaS platform** where users design workflows in the cloud, but execution happens on their own infrastructure!

## 📂 What You Have

### 1. **SaaS Backend** (`/web/`)
Flask API that stores workflows and manages users. Already running on port 5001.

### 2. **Lovable Frontend** (`/lovable-frontend/`)
React app for workflow design. Ready to import to Lovable.

### 3. **CLI Package** (`/zenthral-cli/`)
Python package users install to execute workflows locally.

### 4. **Agent Scripts** (`/scripts/` & `/industries/`)
17 working agents that users can run via the CLI.

## 🚀 Quick Deploy (5 Minutes)

### Step 1: Deploy Backend

```bash
cd web/
railway login
railway init
railway up

# Get your backend URL
railway status
# Example: https://zenthral-production.up.railway.app
```

### Step 2: Deploy Frontend

**Option A: Lovable (Easiest)**
1. Go to Lovable.dev
2. Import `/lovable-frontend/`
3. Set env var: `VITE_API_URL` = your backend URL
4. Deploy!

**Option B: Vercel**
```bash
cd lovable-frontend/
vercel
```

### Step 3: Test Locally (Before Publishing CLI)

```bash
cd zenthral-cli/
pip install -e .
zenthral --help
```

### Step 4: Publish CLI to PyPI (When Ready)

```bash
cd zenthral-cli/
python setup.py sdist bdist_wheel
pip install twine
twine upload dist/*

# Now users can: pip install zenthral-cli
```

## 🎯 How It Works

### 1. User Signs Up
- Goes to your frontend (e.g., `https://zenthral.vercel.app`)
- Creates account
- Gets access to workflow designer

### 2. User Creates Workflow
- Uses natural language: "Every day, scrape HackerNews and email me a summary"
- AI generates workflow JSON
- Workflow stored in your cloud database

### 3. User Installs CLI
```bash
pip install zenthral-cli
zenthral login
# Opens browser to authenticate
```

### 4. User Sets Their API Keys (Local Only)
```bash
export OPENAI_API_KEY=sk-xxxxx
export ANTHROPIC_API_KEY=sk-ant-xxxxx
```

**IMPORTANT**: Keys stay on user's machine, never sent to your server!

### 5. User Runs Workflows
```bash
# Run once
zenthral run

# Or run as daemon
zenthral daemon start
```

CLI polls your SaaS, gets workflows, executes locally, reports back.

## 📁 File Overview

### Key Files to Understand:

**Backend** (`/web/`):
- `app.py` - Main Flask app
- `models/` - Database models (users, workflows, executions)
- `api/v1/` - REST API endpoints
- `services/ai_router.py` - Multi-model AI router
- `services/workflow_engine.py` - Natural language → JSON

**Frontend** (`/lovable-frontend/`):
- `src/App.tsx` - Main React app
- `src/lib/api.ts` - API client
- `src/pages/CreateAutomationPage.tsx` - Workflow designer
- `src/pages/SettingsPage.tsx` - BYOK settings

**CLI** (`/zenthral-cli/`):
- `zenthral/cli.py` - Entry point
- `zenthral/core/client.py` - SaaS API client
- `zenthral/core/executor.py` - Local workflow executor
- `setup.py` - PyPI package config

**Agents** (`/scripts/` & `/industries/`):
- 17 ready-to-use Python agents
- Bundled with CLI
- Users can run them via workflows
